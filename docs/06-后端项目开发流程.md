## 后端技术选型

### 选择Node.js + Express的原因

1. **统一技术栈**：前后端都使用JavaScript/TypeScript
2. **开发效率高**：语法简洁，快速开发
3. **npm生态丰富**：海量第三方包可用
4. **非阻塞I/O**：适合处理并发请求
5. **学习曲线平缓**：容易上手

### 数据存储方案

本项目使用JSON文件存储数据，而不是传统数据库。

**选择JSON文件存储的原因：**
- 简单易用，无需安装配置数据库
- 数据结构灵活
- 适合小型个人网站
- 容易备份和迁移
- 直接可读，方便调试

**限制：**
- 不适合高并发场景
- 不支持复杂查询
- 数据量不能太大（建议小于1000条记录）

## 数据服务层开发

数据服务层负责所有数据的读写操作。

### 创建数据服务文件

在src/services目录创建dataService.js文件。

### 基础功能

数据服务需要提供以下基础功能：

**1. 读取JSON文件**
从指定路径读取JSON文件内容并解析。

**2. 写入JSON文件**
将数据写入JSON文件，自动格式化。

**3. 生成唯一ID**
为新数据生成唯一标识符，可以使用时间戳或UUID。

**4. 获取所有数据**
读取某个数据文件的所有记录。

**5. 根据ID查找**
在数据列表中查找指定ID的记录。

**6. 添加数据**
在数据列表中添加新记录，自动生成ID和时间戳。

**7. 更新数据**
根据ID更新现有记录，保留未修改的字段。

**8. 删除数据**
根据ID删除记录。

## 身份认证实现

### JWT Token认证流程

1. 用户提交用户名和密码
2. 后端验证密码
3. 验证成功生成JWT Token返回给前端
4. 前端保存Token
5. 后续请求在Header中携带Token
6. 后端中间件验证Token有效性
7. 验证通过允许访问，否则返回401错误

### 创建认证中间件

在src/middleware目录创建auth.js文件。

**中间件功能：**
- 从请求头获取Token
- 验证Token是否有效
- 如果有效，将用户信息附加到请求对象
- 如果无效，返回401错误

### 登录路由

在src/routes目录创建auth.js文件。

**登录接口功能：**
- 接收用户名和密码
- 查询users.json文件
- 使用bcrypt比对密码
- 密码正确生成JWT Token返回
- 密码错误返回401错误

### 初始化管理员账号

创建初始化脚本，生成默认管理员账号。

在src目录创建init.js文件：
- 检查users.json是否存在
- 如果不存在，创建默认管理员账号
- 密码使用bcrypt加密后存储

**默认账号：**
- 用户名：admin
- 密码：admin123（首次登录后应修改）

## API路由开发

### 路由结构

每个数据类型对应一个路由文件：

**认证相关：**
- /api/auth/login：登录接口

**内容管理：**
- /api/projects：项目管理
- /api/articles：文章管理
- /api/skills：技能管理
- /api/photos：照片管理
- /api/documents：文档管理
- /api/media：影音书籍
- /api/news：新闻动态
- /api/experiences：工作经历

**网站配置：**
- /api/site-config：网站基本配置
- /api/navigation：导航菜单
- /api/friend-links：友情链接
- /api/footer-settings：页脚设置

**其他：**
- /api/upload：文件上传
- /api/messages：留言管理
- /api/stats：统计数据

### 标准CRUD接口

每个资源通常需要以下接口：

**1. 获取列表（GET）**
- 路径：/api/资源名
- 功能：返回所有记录
- 是否需要认证：前台访问不需要，某些私密内容可能需要

**2. 获取单条（GET）**
- 路径：/api/资源名/:id
- 功能：返回指定ID的记录
- 是否需要认证：通常不需要

**3. 创建（POST）**
- 路径：/api/资源名
- 功能：创建新记录
- 是否需要认证：需要

**4. 更新（PUT）**
- 路径：/api/资源名/:id
- 功能：更新指定ID的记录
- 是否需要认证：需要

**5. 删除（DELETE）**
- 路径：/api/资源名/:id
- 功能：删除指定ID的记录
- 是否需要认证：需要

### 项目管理路由示例

在src/routes创建projects.js文件。

**实现的接口：**

**获取所有项目：**
- GET /api/projects
- 返回所有项目列表
- 前台访问时过滤掉私密项目

**获取单个项目：**
- GET /api/projects/:id
- 返回指定项目详情

**创建项目：**
- POST /api/projects
- 需要认证
- 接收项目数据，保存到projects.json

**更新项目：**
- PUT /api/projects/:id
- 需要认证
- 更新指定项目的信息

**删除项目：**
- DELETE /api/projects/:id
- 需要认证
- 删除指定项目

## 文件上传处理

### 配置Multer

Multer是处理multipart/form-data文件上传的中间件。

在src/routes创建upload.js文件。

**配置内容：**
- 上传目录：uploads文件夹
- 文件命名：时间戳+原始文件名，避免冲突
- 文件大小限制：最大50MB

### 图片上传接口

**功能需求：**
- 接收图片文件
- 验证文件类型（只允许图片）
- 保存原图到uploads目录
- 使用Sharp生成缩略图
- 返回图片URL

**接口路径：** POST /api/upload/image

**返回数据：**
- 原图URL
- 缩略图URL

### 文件上传接口

**功能需求：**
- 接收各种类型文件
- 验证文件大小
- 保存到uploads目录
- 返回文件URL

**接口路径：** POST /api/upload/file

**支持的文件类型：**
- 图片：jpg、png、gif、webp
- 文档：pdf、doc、docx、xls、xlsx
- 音频：mp3、wav
- 视频：mp4

### Sharp图片处理

Sharp用于图片压缩和缩略图生成。

**功能：**
- 压缩图片大小
- 生成缩略图（宽度800px，保持比例）
- 转换格式
- 优化质量

**处理流程：**
1. 接收上传的图片
2. 保存原图
3. 使用Sharp压缩生成缩略图
4. 保存缩略图到thumbnails目录
5. 返回两个URL

## 错误处理

### 统一错误响应格式

所有错误返回统一格式的JSON：
- 包含error字段，描述错误信息
- 使用合适的HTTP状态码

### 常见错误状态码

- 400：Bad Request，请求参数错误
- 401：Unauthorized，未登录或Token无效
- 403：Forbidden，没有权限
- 404：Not Found，资源不存在
- 500：Internal Server Error，服务器内部错误

### 错误处理最佳实践

1. **参数验证**：检查必填参数是否存在
2. **数据验证**：验证数据格式是否正确
3. **捕获异常**：使用try-catch捕获所有可能的错误
4. **记录日志**：将错误信息记录到日志文件
5. **友好提示**：返回清晰的错误信息给前端

## 背景音乐功能开发

这是一个特别的功能点，记录完整的开发过程。

### 功能需求

- 支持上传多首歌曲
- 创建播放列表
- 自动循环播放
- 可以切换上一首/下一首
- 控制音量
- 页面刷新后自动播放

### 后端实现

**数据结构修改：**
将site-config.json中的music配置从单个musicUrl改为musicList数组：
- enabled：是否启用音乐
- musicList：数组，每个元素包含url和name
- volume：音量大小（0-1）

**API接口：**
- GET /api/site-config/music：获取音乐配置
- PUT /api/site-config/music：更新音乐配置

### 前端实现

**组件功能：**
- 显示当前播放的歌曲名称
- 显示播放/暂停按钮
- 显示上一首/下一首按钮
- 显示音量控制滑块
- 显示播放进度（当前第几首/总共几首）
- 歌曲播放结束自动播放下一首
- 使用useState管理播放状态和当前歌曲索引

**音乐播放器位置：**
固定在页面右下角，不随滚动消失。

### 遇到的问题和解决

**问题1：管理后台背景音乐页面空白**

**原因：**
- 代码中访问settings.musicList.length
- 但API返回的数据可能没有musicList字段
- 导致undefined.length报错，页面崩溃

**解决方法：**
在前端代码中添加默认值处理，如果musicList不存在就设置为空数组。

**问题2：文件上传413错误**

**原因：**
音乐文件较大，超过了body-parser和Nginx的请求体大小限制。

**解决方法：**
1. 修改backend的body-parser限制为50MB
2. 修改Nginx配置，增加client_max_body_size为50M
3. 需要在两个地方修改：
   - Nginx全局配置
   - location /api块中也要添加

## 常见问题

### 问题1：端口被占用

**症状：** 启动服务器时报错端口已被使用

**解决方法：**
- Windows：查找并终止占用端口的进程
- Mac/Linux：使用kill命令终止进程
- 或者修改.env文件使用其他端口

### 问题2：JSON文件损坏

**症状：** 读取数据时报错JSON解析失败

**原因：** JSON文件格式错误或被损坏

**解决方法：**
- 使用JSON验证工具检查文件格式
- 恢复备份文件
- 手动修复JSON格式错误

### 问题3：文件上传失败

**症状：** 上传文件时报错或没有响应

**排查步骤：**
- 检查uploads目录是否存在且有写权限
- 检查文件大小是否超过限制
- 检查文件类型是否允许
- 查看服务器日志错误信息

### 问题4：Token验证失败

**症状：** 已登录但请求仍返回401错误

**原因：**
- Token过期
- Token格式不正确
- 请求头没有正确携带Token

**解决方法：**
- 检查Token是否正确保存和携带
- 检查JWT_SECRET是否一致
- 重新登录获取新Token
