# 第1部分：项目概述和准备工作

## 项目简介

这是一个完整的全栈个人作品集网站项目，用于展示个人的项目经验、技能、博客文章、照片等内容。网站分为前台展示和后台管理两个部分，支持动态内容管理。

### 项目基本信息

- **项目名称**：个人作品集网站
- **开发时间**：2025年12月
- **网站域名**：www.bohenan.com
- **项目类型**：全栈Web应用
- **开发模式**：个人独立开发

---

## 网站功能介绍

### 前台展示功能

网站前台是面向访客的展示页面，包含以下主要功能：

1. **首页展示**
   - 显示个人基本信息，包括头像、姓名、职位、联系方式
   - 照片墙自动轮播展示，每6秒切换一次
   - 响应式设计，自动适配手机和平板设备

2. **内容展示模块**
   - 博客文章：展示技术文章，支持分类和标签筛选
   - 新闻动态：展示个人最新动态和成就
   - 项目经验：详细展示项目信息，包括技术栈、项目金额、项目图片和相关文档
   - 技能展示：按分类展示个人技能树，带有熟练度评级
   - 影音书籍：展示看过的电影和书籍，包含评分和个人评论
   - 精彩瞬间：瀑布流照片墙展示，支持灯箱效果预览大图
   - 知识文档：技术文档展示，支持文档附件下载
   - 工作经历：时间轴方式展示工作履历

3. **特色功能**
   - 背景音乐播放器：支持多首歌曲循环播放，可以切换上一首和下一首
   - 在线联系表单：访客可以直接留言
   - 友情链接：底部展示友情链接
   - 图片懒加载：优化页面加载速度
   - SEO优化：支持搜索引擎收录和社交媒体分享

### 后台管理功能

后台管理系统用于管理网站的所有内容：

1. **内容管理**
   - 项目管理：添加、编辑、删除项目，上传项目图片和附件，可以设置私密项目
   - 技能管理：按分类管理技能，设置熟练度等级
   - 博客管理：使用Markdown编辑器撰写文章，管理分类和标签
   - 新闻管理：发布新闻动态，可以设置置顶
   - 照片管理：支持单张和批量上传照片，按分类管理
   - 文档管理：上传各类文档附件，最大支持50MB
   - 影音书籍管理：添加电影或书籍评价，上传封面图片

2. **网站配置**
   - 导航菜单管理：自定义网站导航栏
   - 友情链接管理：添加友情链接，上传logo图片
   - 页脚设置：自定义底部版权和联系信息
   - 社交媒体管理：配置各种社交媒体链接
   - 标签管理：统一管理所有标签
   - 背景音乐设置：上传音乐文件，创建播放列表
   - 个人信息管理：编辑个人资料，上传首页照片墙图片

3. **数据统计**
   - 访问统计：查看各个页面的访问量
   - 数据概览：统计各类内容的数量
   - 留言管理：查看和回复用户留言

## 技术栈选择

### 前端技术

这个项目的前端使用了现代化的技术栈：

1. **核心框架**
   - React 18：用于构建用户界面的JavaScript库
   - TypeScript：提供类型安全，减少开发错误
   - Vite：新一代前端构建工具，开发速度快

2. **UI和样式**
   - Tailwind CSS：实用优先的CSS框架，快速构建界面
   - React Masonry CSS：实现瀑布流布局
   - 自定义懒加载组件：优化图片加载性能

3. **路由和状态**
   - React Router v6：处理页面路由
   - React Hooks：使用useState、useEffect等管理组件状态

4. **工具库**
   - Axios：处理HTTP请求
   - React Markdown：渲染Markdown格式的文章
   - date-fns：处理日期格式化

### 后端技术

后端采用Node.js生态系统：

1. **核心框架**
   - Node.js 18：JavaScript运行环境
   - Express.js：轻量级Web应用框架

2. **数据存储**
   - JSON文件存储：使用JSON文件作为简单数据库
   - 文件系统：使用Node.js的fs模块进行文件操作

3. **安全认证**
   - JWT：用户身份认证令牌
   - bcryptjs：密码加密存储
   - CORS：配置跨域资源共享

4. **文件处理**
   - Multer：处理文件上传
   - Sharp：图片压缩和缩略图生成

### 部署技术

项目部署使用了标准的Web服务器配置：

1. **服务器环境**
   - Ubuntu 20.04 LTS操作系统
   - Nginx作为Web服务器和反向代理
   - PM2进程管理器，保证后端服务稳定运行

2. **域名和SSL**
   - 从域名注册商购买域名
   - 使用云服务商的DNS解析服务
   - Let's Encrypt免费SSL证书，实现HTTPS访问

3. **云服务器**
   - 可选择Google Cloud、阿里云、腾讯云等
   - 推荐配置：2核CPU、4GB内存、40GB硬盘
   - 需要固定公网IP地址
   - 开放80和443端口用于HTTP和HTTPS访问

## 开发前的准备工作

### 本地开发环境准备

在开始开发之前，需要准备以下工具：

1. **必需软件**
   - Node.js：版本20或以上，建议使用nvm管理Node.js版本
   - npm或yarn：包管理工具，npm随Node.js自动安装
   - Git：版本控制工具
   - 代码编辑器：推荐使用VS Code

2. **VS Code推荐插件**
   - ESLint：JavaScript代码检查
   - Prettier：代码格式化工具
   - Tailwind CSS IntelliSense：Tailwind自动补全
   - GitLens：增强Git功能

3. **电脑配置建议**
   - 操作系统：Windows 10/11、macOS或Linux都可以
   - 内存：至少8GB
   - 硬盘：至少10GB可用空间

### 生产环境准备

部署到线上需要准备：

1. **云服务器**
   - 操作系统：Ubuntu 20.04 LTS或更高版本
   - 配置：最低2核CPU、4GB内存、40GB硬盘
   - 网络：需要固定的公网IP地址

2. **域名**
   - 购买一个域名（如.com、.cn等）
   - 完成域名实名认证
   - 配置DNS解析

3. **其他准备**
   - 一张外币信用卡（如果使用国外云服务商）
   - 或支付宝/微信（如果使用国内云服务商）

6. 测试线上功能是否正常

1. 收集用户反馈和新需求

## 项目目录结构说明

项目采用前后端分离的结构：

### 前端目录

前端代码主要在src文件夹中：

- components文件夹：存放所有React组件，包括导航栏、页脚、音乐播放器、管理后台组件等
- pages文件夹：存放页面级组件，如首页、登录页、管理后台页面等
- services文件夹：封装API调用函数
- types文件夹：TypeScript类型定义
- utils文件夹：通用工具函数

构建后会生成dist文件夹，里面是要部署到服务器的静态文件。

### 后端目录

后端代码在backend文件夹中：

- src/controllers：控制器，处理业务逻辑
- src/middleware：中间件，如身份验证
- src/routes：API路由定义
- src/services：数据服务，处理数据读写
- data文件夹：JSON数据文件存储
- uploads文件夹：用户上传的文件存储

### 文档目录

docs文件夹存放项目文档，记录开发过程和遇到的问题。

# 第2部分：云服务器选择和购买

## 为什么需要云服务器

开发个人网站需要一台24小时运行的服务器来托管网站。云服务器相比传统服务器有以下优势：

1. **按需付费**：根据使用时间和配置付费，不需要一次性购买硬件
2. **快速部署**：几分钟内就能创建一台服务器
3. **稳定可靠**：云服务商提供高可用性保障
4. **弹性扩展**：随时可以升级配置
5. **全球部署**：可以选择离用户最近的机房

## Google Cloud购买流程

### 第一步：注册账号

1. 访问Google Cloud官网
2. 点击右上角的"开始免费试用"按钮
3. 使用Google账号登录（如果没有需要先注册Google账号）
4. 填写账单信息和信用卡信息进行验证
5. 成功后获得300美元免费额度，有效期3个月

**注意事项：**
- 需要外币信用卡（Visa或MasterCard）
- 信用卡会扣除1美元用于验证，之后会退还
- 免费额度用完或到期后才会开始计费

### 第二步：创建项目

1. 登录Google Cloud控制台
2. 点击顶部的项目选择器
3. 点击"新建项目"
4. 输入项目名称，比如"personal-portfolio"
5. 点击"创建"按钮

**说明：** Google Cloud使用项目来组织资源，一个账号可以创建多个项目。

### 第三步：创建虚拟机实例

1. 在左侧菜单找到"Compute Engine"（计算引擎）
2. 点击"VM实例"（虚拟机实例）
3. 点击"创建实例"按钮

**配置虚拟机：**

**基本信息：**
- 实例名称：输入一个好记的名称，比如"portfolio-server"
- 区域：选择离你的目标用户最近的区域
  - 如果面向中国用户，选择asia-east1（台湾）或asia-northeast1（日本）
  - 如果面向全球用户，选择us-central1（美国中部）

**机器配置：**
- 系列：选择E2（性价比高的通用型）
- 机器类型：选择e2-medium（2个vCPU，4GB内存）

**启动磁盘：**
- 点击"更改"按钮
- 操作系统：选择Ubuntu
- 版本：选择Ubuntu 20.04 LTS
- 磁盘类型：标准永久性磁盘
- 大小：40GB

**防火墙：**
- 勾选"允许HTTP流量"
- 勾选"允许HTTPS流量"

完成配置后点击"创建"按钮，等待几分钟虚拟机就会创建完成。

### 第四步：配置静态IP

默认情况下，虚拟机的IP地址是临时的，重启后会改变。我们需要将其改为静态IP：

1. 在左侧菜单找到"VPC网络"
2. 点击"外部IP地址"
3. 找到刚创建的虚拟机对应的IP地址
4. 类型那一列会显示"临时"，点击它
5. 选择"静态"
6. 输入一个名称，比如"portfolio-static-ip"
7. 点击"预留"按钮

**注意：** 静态IP会产生少量额外费用（约每月5-10美元），但对于网站来说是必需的，因为域名需要指向一个固定的IP。

### 第五步：记录服务器信息

创建完成后，记录以下信息：

- 外部IP地址：这是访问服务器的公网IP
- 实例名称：管理时需要用到
- 区域：了解服务器所在位置

## 阿里云购买流程

### 第一步：注册账号

1. 访问阿里云官网
2. 点击"免费注册"
3. 使用手机号完成注册
4. 登录后完成实名认证（上传身份证照片）

**注意：** 实名认证是必需的，通常1-2小时内审核完成。

### 第二步：购买ECS实例

1. 在阿里云官网搜索"云服务器ECS"
2. 点击"立即购买"
3. 进入配置页面

**购买配置：**

**计费方式：**
- 包年包月：适合长期使用，价格优惠
- 按量付费：按小时计费，灵活但总价更高
- 建议选择包年包月，购买1年

**地域和可用区：**
- 如果面向全国用户，选择华东、华北或华南
- 如果主要是本地用户，选择离你最近的地域
- 可用区选择随机分配即可

**实例规格：**
- 实例类型：选择"共享标准型"或"计算型"
- 规格：2核4GB（够用且性价比高）

**镜像：**
- 选择"公共镜像"
- 操作系统：Ubuntu
- 版本：Ubuntu 20.04 64位

**存储：**
- 系统盘：高效云盘，40GB

**网络：**
- 分配公网IP：勾选
- 带宽计费模式：按固定带宽
- 带宽：3Mbps或5Mbps

**安全组：**
- 使用默认安全组，或者新建安全组
- 确保开放以下端口：
  - 22端口（SSH远程连接）
  - 80端口（HTTP访问）
  - 443端口（HTTPS访问）
  - 3001端口（后端API，可选）

### 第三步：设置密码

- 设置实例名称：比如"portfolio-server"
- 设置root密码：务必使用强密码并妥善保存
- 主机名：可以使用默认值

### 第四步：确认订单

1. 勾选同意服务协议
2. 确认配置信息
3. 选择购买时长（建议1年）
4. 查看优惠券，新用户通常有折扣
5. 确认支付金额
6. 使用支付宝或微信支付

### 第五步：查看实例信息

支付成功后：

1. 进入阿里云控制台
2. 找到"云服务器ECS"
3. 点击"实例列表"
4. 可以看到刚购买的服务器

**记录以下信息：**
- 公网IP地址
- 实例ID
- root密码
- 区域信息

## 腾讯云购买流程

腾讯云的购买流程与阿里云非常相似：

### 第一步：注册和实名认证

1. 访问腾讯云官网
2. 注册账号（手机号或微信）
3. 完成实名认证

### 第二步：选择云服务器CVM

1. 进入云服务器CVM产品页
2. 点击"立即选购"
3. 选择配置（与阿里云类似）

**推荐配置：**
- 地域：根据目标用户选择
- 机型：标准型S5，2核4GB
- 镜像：Ubuntu Server 20.04
- 系统盘：50GB云硬盘
- 公网带宽：按流量计费或固定带宽3Mbps

### 第三步：设置安全组

确保安全组规则包含：
- 放通22端口（SSH）
- 放通80端口（HTTP）
- 放通443端口（HTTPS）

### 第四步：完成购买

设置实例密码，确认订单并支付。

## 服务器购买后的检查

无论选择哪个云服务商，购买完成后需要检查：

### 1. 确认服务器状态

在控制台查看实例状态是否为"运行中"。

### 2. 记录关键信息

- 公网IP地址（最重要）
- SSH登录用户名（通常是root或ubuntu）
- SSH登录密码
- 服务器所在区域

### 3. 测试网络连通性

在本地电脑打开命令行工具，尝试ping服务器IP：

在Windows：打开命令提示符
在Mac/Linux：打开终端

输入命令测试（将IP替换为你的服务器IP）：
ping 你的服务器IP

如果能收到回复，说明网络连通正常。

### 4. 检查防火墙规则

确认云服务商的安全组或防火墙已经开放了必要的端口。

## 常见问题

### 问题1：无法连接到服务器

**可能原因：**
- 安全组或防火墙没有开放22端口
- 输入的IP地址不正确
- 本地网络问题

**解决方法：**
- 检查云服务商控制台的安全组配置
- 确认IP地址是否正确
- 尝试使用手机热点连接测试

### 问题2：忘记root密码

**解决方法：**
- 在云服务商控制台可以重置密码
- 阿里云：实例列表 → 更多 → 密码/密钥 → 重置密码
- GCP：需要使用SSH密钥登录，可以在控制台生成密钥

### 问题3：如何选择服务器地域

**建议：**
- 如果用户主要在中国，选择国内地域（但需要备案）
- 如果不想备案，可以选择香港或海外地域
- 如果追求访问速度，选择离用户最近的地域

### 问题4：配置不够用怎么办

**解决方法：**
- 大多数云服务商支持在线升级配置
- 阿里云和腾讯云可以直接升级
- GCP可以停止实例后更改机器类型

## 成本估算

### 国内云服务商（阿里云/腾讯云）

**最低配置：**
- 1核2GB，1Mbps带宽
- 价格：约60-80元/月

**推荐配置：**
- 2核4GB，3-5Mbps带宽
- 价格：约120-180元/月
- 新用户首年通常有5-7折优惠

### Google Cloud Platform

**推荐配置：**
- e2-medium（2核4GB）
- 40GB磁盘
- 价格：约30-40美元/月（约210-280元人民币）
- 新用户有300美元免费额度，可以用6-10个月

### 节省成本的技巧

1. **利用新用户优惠**：首次购买通常有大幅折扣
2. **包年购买**：比按月购买便宜20-30%
3. **关注促销活动**：双十一、618等活动期间折扣力度大
4. **合理选择带宽**：带宽是费用大头，按需选择即可
