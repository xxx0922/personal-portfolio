## 前端技术选型

### 选择React的原因

1. **组件化开发**：将页面拆分成独立的组件，代码复用性高
2. **虚拟DOM**：性能优秀，更新效率高
3. **生态丰富**：有大量第三方库和工具
4. **TypeScript支持**：提供类型检查，减少错误
5. **社区活跃**：遇到问题容易找到解决方案

### 选择Vite的原因

1. **启动速度快**：秒级启动，开发体验好
2. **热更新快**：修改代码后瞬间看到效果
3. **开箱即用**：无需复杂配置
4. **现代化**：原生支持ES模块

### 选择Tailwind CSS的原因

1. **快速开发**：使用工具类快速构建界面
2. **响应式设计**：内置响应式断点
3. **可定制**：容易自定义主题
4. **文件体积小**：自动删除未使用的样式

## 项目初始化

### 创建React项目

在本地电脑上选择一个工作目录，打开终端执行创建命令。

Vite会询问几个问题：
1. **项目名称**：输入项目名称，比如"personal-portfolio"
2. **框架选择**：选择React
3. **语言选择**：选择TypeScript

项目创建完成后，进入项目目录。

### 安装依赖

安装项目依赖包，这个过程需要几分钟。

### 启动开发服务器

安装完成后，启动本地开发服务器。

浏览器访问提示的地址（通常是localhost:5173），应该能看到Vite的欢迎页面。

### 安装Tailwind CSS

#### 第一步：安装Tailwind相关包

安装Tailwind CSS及其依赖。

#### 第二步：初始化Tailwind配置

生成Tailwind配置文件。

#### 第三步：配置Tailwind

编辑生成的tailwind.config.js文件，配置要扫描的文件路径，让Tailwind知道在哪些文件中查找要使用的类名。

#### 第四步：引入Tailwind样式

在src/index.css文件中引入Tailwind的基础样式、组件样式和工具样式。

## 项目结构规划

一个清晰的项目结构有助于后期维护。

### 目录结构说明

**src/components/**
存放所有可复用的React组件，比如：
- Navbar.tsx：导航栏组件
- Footer.tsx：页脚组件
- BackgroundMusic.tsx：背景音乐播放器
- LazyImage.tsx：懒加载图片组件

**src/pages/**
存放页面级组件，比如：
- HomePage.tsx：首页
- AdminPage.tsx：管理后台
- LoginPage.tsx：登录页
- ProjectDetailPage.tsx：项目详情页

**src/services/**
存放API调用相关的代码，封装所有与后端的通信逻辑。

**src/types/**
存放TypeScript类型定义，定义数据结构的接口。

**src/utils/**
存放工具函数，比如日期格式化、字符串处理等。

**src/hooks/**
存放自定义React Hooks，比如useSEO用于设置页面SEO信息。

## 前端开发流程

### 第一步：设计组件结构

在开始编码前，先规划好需要哪些组件。

#### 公共组件
- 导航栏：显示网站菜单
- 页脚：显示版权和友情链接
- 背景音乐播放器：控制音乐播放
- 图片懒加载组件：优化图片加载

#### 页面组件
- 首页：展示个人信息和照片墙
- 项目列表页：展示所有项目
- 项目详情页：展示单个项目的详细信息
- 博客列表页：展示文章列表
- 文章详情页：显示文章内容
- 管理后台页：管理网站内容
- 登录页：管理员登录

### 第二步：创建基础组件

#### 导航栏组件

导航栏是网站的核心组件之一，需要实现：
- 显示网站logo和标题
- 显示导航菜单
- 响应式设计，移动端显示汉堡菜单
- 高亮当前页面
- 滚动时可能改变样式（可选）

**开发要点：**
- 使用React Router的Link组件实现导航
- 使用useState管理移动菜单的展开/收起状态
- 使用Tailwind的响应式类实现自适应布局
- 可以添加滚动监听改变导航栏样式

#### 页脚组件

页脚显示版权信息、友情链接、社交媒体链接等。

**开发要点：**
- 布局使用Flexbox或Grid
- 友情链接从API获取
- 显示当前年份（动态获取）
- 包含网站备案号（如果需要）

#### 图片懒加载组件

懒加载可以显著提升页面性能，只在图片进入视口时才加载。

**实现思路：**
- 使用Intersection Observer API监听元素是否进入视口
- 初始显示占位符或模糊图
- 进入视口后加载真实图片
- 加载完成后显示图片并添加淡入动画

**开发要点：**
- 组件接收图片URL、alt文本、占位符等props
- 使用useRef引用DOM元素
- 使用useEffect设置Observer
- 使用useState管理加载状态

### 第三步：创建页面组件

#### 首页开发

首页是访客的第一印象，需要重点设计。

**主要内容：**
- 个人头像和基本信息
- 一句话介绍
- 照片墙轮播（多张照片自动切换）
- 技能标签云
- 最新项目展示
- 联系方式

**开发要点：**
- 使用useState和useEffect实现照片轮播
- 设置定时器自动切换照片（每6秒）
- 添加切换动画效果
- 响应式布局，适配各种屏幕尺寸

#### 项目列表页

展示所有项目卡片。

**功能需求：**
- 网格布局展示项目卡片
- 每个卡片显示项目缩略图、标题、简介、技术栈
- 可以按技术栈或分类筛选
- 点击卡片进入项目详情页
- 私密项目不显示（需要登录后台查看）

**开发要点：**
- 从API获取项目列表数据
- 使用Grid布局展示卡片
- 添加筛选功能（可选）
- 使用React Router导航到详情页

#### 项目详情页

显示单个项目的完整信息。

**显示内容：**
- 项目名称和标语
- 详细描述
- 技术栈标签
- 项目角色和工作内容
- 项目图片轮播
- 项目附件下载
- 项目开始和结束时间
- 合同金额（可选）

**开发要点：**
- 从URL获取项目ID
- 调用API获取项目详情
- 实现图片轮播或画廊效果
- 处理附件下载链接

#### 博客列表页

展示所有文章。

**功能需求：**
- 列表或卡片形式展示文章
- 显示标题、摘要、发布时间、分类、标签
- 可以按分类或标签筛选
- 支持搜索功能（可选）
- 点击进入文章详情页

**开发要点：**
- 从API获取文章列表
- 实现筛选和搜索逻辑
- 处理日期格式显示
- 显示阅读次数和点赞数（如果有）

#### 文章详情页

显示文章完整内容。

**显示内容：**
- 文章标题
- 发布时间和作者
- 分类和标签
- 文章正文（Markdown渲染）
- 相关文章推荐（可选）
- 上一篇/下一篇导航

**开发要点：**
- 使用React Markdown渲染Markdown内容
- 处理代码高亮（可使用highlight.js）
- 添加目录导航（可选）
- 实现评论功能（可选）

### 第四步：开发管理后台

管理后台是网站的核心功能，用于管理所有内容。

#### 登录页面

**功能需求：**
- 用户名和密码输入框
- 登录按钮
- 表单验证
- 登录后跳转到管理后台
- 保存登录状态（使用JWT Token）

**开发要点：**
- 表单提交前进行前端验证
- 调用登录API获取Token
- 将Token保存到localStorage
- 登录成功后跳转到管理后台
- 处理登录失败提示

#### 后台主页面

**布局结构：**
- 顶部导航栏：显示网站名称、退出登录按钮
- 左侧菜单栏：列出所有管理功能
- 右侧内容区：显示当前选中的管理模块

**功能模块：**
- 仪表板：显示统计数据
- 项目管理：增删改查项目
- 文章管理：增删改查文章
- 照片管理：上传和管理照片
- 技能管理：管理技能列表
- 文档管理：上传文档附件
- 音乐设置：配置背景音乐
- 个人信息：编辑个人资料
- 网站配置：配置导航、页脚、友情链接等

**开发要点：**
- 使用路由实现不同模块的切换
- 每个请求都需要携带Token
- Token过期自动跳转到登录页
- 左侧菜单高亮当前模块

#### 内容管理组件

每个内容类型都需要一个管理组件。

**通用功能：**
- 列表显示：表格形式展示所有数据
- 添加按钮：打开添加表单
- 编辑按钮：打开编辑表单，填充现有数据
- 删除按钮：确认后删除数据
- 搜索功能：快速查找内容

**表单功能：**
- 输入验证：必填项、格式验证
- 文件上传：图片、音频、文档
- 富文本编辑：文章内容编辑
- 标签选择：多选标签
- 日期选择：日期时间选择器

**开发要点：**
- 表单组件可以复用，传入不同配置
- 上传文件需要使用FormData
- 处理上传进度显示
- 上传成功后刷新列表
- 错误处理和用户提示

## API服务封装

为了方便调用后端API，需要封装一个API服务层。

### 创建API基础配置

在src/services目录创建API服务文件。

**功能包括：**
- 配置API基础URL
- 封装所有API调用函数
- 统一处理Token
- 统一处理错误

### API函数示例

**获取项目列表：**
发送GET请求获取所有项目数据。

**创建项目：**
发送POST请求创建新项目，携带项目数据。

**上传文件：**
发送POST请求上传文件，使用FormData格式。

**登录：**
发送POST请求，传入用户名和密码，返回Token。

### Token管理

**保存Token：**
登录成功后保存Token到localStorage。

**携带Token：**
每个需要认证的请求都在请求头中携带Token。

**清除Token：**
退出登录或Token过期时清除。

## 响应式设计

网站需要在各种设备上都能正常显示。

### Tailwind响应式断点

Tailwind提供了内置的响应式断点：
- sm：640px以上（小屏幕，手机横屏）
- md：768px以上（平板）
- lg：1024px以上（小笔记本）
- xl：1280px以上（桌面显示器）
- 2xl：1536px以上（大屏显示器）

### 响应式布局策略

**移动优先：**
先设计移动端布局，然后逐步增强到大屏幕。

**导航栏：**
- 移动端：汉堡菜单，点击展开
- 桌面端：水平菜单

**内容布局：**
- 移动端：单列布局
- 平板：两列布局
- 桌面：三列或四列布局

**图片：**
- 移动端：全宽显示
- 桌面：固定宽度或比例

### 测试响应式

开发时需要在不同尺寸下测试：
- 使用浏览器开发者工具的设备模拟
- 测试手机竖屏（375px）
- 测试手机横屏（667px）
- 测试平板（768px）
- 测试桌面（1280px）

## 性能优化

### 图片优化

1. **使用懒加载**：只加载可见区域的图片
2. **压缩图片**：上传前压缩图片大小
3. **使用WebP格式**：体积更小，质量更好
4. **响应式图片**：不同尺寸设备加载不同大小的图片

### 代码分割

1. **路由懒加载**：页面组件按需加载
2. **组件懒加载**：大型组件延迟加载
3. **第三方库按需引入**：只引入使用的部分

### 其他优化

1. **缓存策略**：合理设置浏览器缓存
2. **减少HTTP请求**：合并资源文件
3. **使用CDN**：静态资源使用CDN加速
4. **删除未使用代码**：生产构建时自动删除

## 常见问题

### 问题1：跨域请求失败

**症状：** 前端请求后端API时出现CORS错误

**原因：** 后端没有配置CORS允许前端域名

**解决方法：** 在后端server.js中配置CORS，允许前端开发服务器地址（localhost:5173）

### 问题2：路由404错误

**症状：** 刷新页面后出现404错误

**原因：** 单页应用路由需要服务器配置

**解决方法：** 开发时使用Vite自动处理，部署后需要配置Nginx

### 问题3：图片路径错误

**症状：** 图片无法显示

**原因：** 图片路径配置错误

**解决方法：** 确保图片URL使用后端完整路径

### 问题4：Token过期未处理

**症状：** Token过期后仍然可以访问管理后台

**解决方法：** 在API请求拦截器中检查401响应，自动跳转到登录页
