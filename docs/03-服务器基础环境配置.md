## 连接到服务器

购买服务器后，第一步是通过SSH远程连接到服务器进行配置。

### Windows用户连接方法

#### 方法一：使用Windows Terminal

Windows 10和Windows 11自带SSH客户端，可以直接使用：

1. 打开开始菜单，搜索"Terminal"或"PowerShell"
2. 在终端中输入连接命令（将IP替换为你的服务器IP）
3. 输入完命令按回车
4. 首次连接会提示是否信任，输入yes回车
5. 然后输入服务器密码（输入时不显示字符，这是正常的）
6. 密码正确后就成功登录了

#### 方法二：使用PuTTY工具

如果是Windows 7或更老的系统：

1. 下载PuTTY工具（免费软件）
2. 安装并打开PuTTY
3. 在Host Name栏输入服务器IP地址
4. Port保持22不变
5. Connection type选择SSH
6. 点击Open按钮
7. 首次连接会弹出安全警告，点击"是"
8. 在黑色终端窗口输入用户名root
9. 输入密码（不显示字符）
10. 登录成功

### Mac用户连接方法

Mac系统自带SSH客户端：

1. 打开"终端"应用（在应用程序的实用工具文件夹中）
2. 输入连接命令
3. 首次连接提示是否继续，输入yes
4. 输入密码
5. 登录成功

### Linux用户连接方法

Linux系统都自带SSH客户端，使用方法与Mac相同，在终端输入连接命令即可。

### 配置SSH密钥登录（推荐）

每次输入密码比较麻烦，可以配置SSH密钥实现免密码登录：

#### 第一步：在本地生成密钥对

在本地电脑的终端执行生成密钥命令。

系统会询问：
- 保存位置：直接按回车使用默认位置
- 密码：可以设置也可以留空（直接回车）

#### 第二步：上传公钥到服务器

将本地生成的公钥内容复制到服务器。

有两种方法：

**方法一：自动上传（推荐）**
使用自动上传命令，输入服务器密码即可完成。

**方法二：手动上传**
1. 在本地查看公钥内容
2. 复制公钥内容
3. 登录服务器
4. 在服务器上创建SSH目录
5. 将公钥内容粘贴到授权文件中
6. 设置正确的文件权限

#### 第三步：测试密钥登录

再次连接服务器时，应该不需要输入密码就能直接登录了。

## 更新系统软件

首次登录服务器后，第一件事是更新系统。

### 为什么要更新系统

- 安装最新的安全补丁
- 修复已知的系统漏洞
- 获得最新的软件包信息
- 提高系统稳定性

### 更新步骤

#### 第一步：更新软件包列表

这个命令会从软件源下载最新的软件包列表，但不会安装任何软件。

#### 第二步：升级已安装的软件

这个命令会将系统中所有已安装的软件升级到最新版本。添加-y参数表示自动确认，不需要手动输入yes。

整个过程可能需要5-15分钟，耐心等待。

#### 第三步：清理不需要的包

升级完成后，清理系统中不再需要的旧软件包，释放磁盘空间。

## 安装Node.js

Node.js是运行后端代码的必需环境。

### 为什么选择Node.js 20

- Node.js 20是LTS（长期支持）版本，稳定可靠
- 性能优秀，支持最新的JavaScript特性
- 生态丰富，npm包资源多

### 安装方法一：使用NodeSource仓库（推荐）

#### 第一步：添加NodeSource仓库

下载并运行NodeSource的安装脚本，会自动添加Node.js 20的软件源。

#### 第二步：安装Node.js

安装Node.js，npm会随Node.js一起安装。

#### 第三步：验证安装

检查Node.js版本，应该显示v20.x.x
检查npm版本，应该显示10.x.x

### 安装方法二：使用nvm（适合需要多版本切换）

如果将来可能需要切换不同的Node.js版本，可以使用nvm（Node Version Manager）：

#### 第一步：安装nvm

下载并安装nvm管理工具。

#### 第二步：重新加载配置

让nvm立即生效。

#### 第三步：安装Node.js 20

使用nvm安装Node.js 20版本。

#### 第四步：设置默认版本

设置Node.js 20为默认版本，这样每次登录都会自动使用这个版本。

## 安装Nginx

Nginx是高性能的Web服务器和反向代理服务器。

### Nginx的作用

1. **Web服务器**：托管前端静态文件（HTML、CSS、JS）
2. **反向代理**：将API请求转发到后端服务
3. **负载均衡**：支持多个后端实例（扩展时使用）
4. **SSL终端**：处理HTTPS加密

### 安装步骤

#### 第一步：安装Nginx

从Ubuntu软件源安装Nginx。

#### 第二步：启动Nginx

安装完成后需要手动启动服务。

#### 第三步：设置开机自启

让Nginx在服务器重启后自动启动。

#### 第四步：检查运行状态

查看Nginx是否正在运行，应该显示"active (running)"。

#### 第五步：验证安装

在浏览器中输入服务器的IP地址，应该能看到Nginx的欢迎页面，显示"Welcome to nginx!"，说明Nginx已成功运行。

## 安装PM2

PM2是Node.js进程管理工具，用于管理后端服务。

### PM2的作用

1. **进程守护**：应用崩溃后自动重启
2. **开机自启**：服务器重启后自动启动应用
3. **日志管理**：自动记录应用日志
4. **性能监控**：实时查看CPU和内存使用情况
5. **零停机重启**：更新代码时不中断服务

### 安装步骤

#### 第一步：全局安装PM2

使用npm全局安装PM2，-g参数表示全局安装。

#### 第二步：验证安装

检查PM2版本，确认安装成功。

#### 第三步：配置开机自启

运行startup命令，PM2会生成一个系统启动脚本命令。

复制PM2输出的命令（通常是一个很长的sudo命令），然后执行这个命令。

这样设置后，服务器重启时PM2会自动启动，并恢复所有之前运行的应用。

## 配置防火墙

防火墙用于保护服务器安全，只允许必要的端口接受外部访问。

### 使用UFW防火墙

Ubuntu自带UFW（Uncomplicated Firewall）防火墙工具，简单易用。

### 第一步：安装UFW

Ubuntu通常已经预装了UFW。

### 第二步：设置默认规则

- 拒绝所有入站连接（从外部连接到服务器）
- 允许所有出站连接（从服务器连接到外部）

### 第三步：允许SSH端口（重要！）

**特别注意：** 在启用防火墙之前，必须先允许SSH端口（22），否则你会被锁在服务器外面，无法再次连接！

### 第四步：启用防火墙

确认允许SSH后，启用UFW防火墙。

系统会提示可能中断现有SSH连接，输入y确认。

### 第五步：查看防火墙状态

检查防火墙状态和规则。

### 开放其他必需端口

#### 开放HTTP端口（80）

允许HTTP访问，用于网站访问。

#### 开放HTTPS端口（443）

允许HTTPS访问，用于加密的网站访问。

#### 开放后端API端口（可选）

如果需要直接访问后端API，可以开放3001端口（后面配置了Nginx反向代理后这个可以不开放）。

### 查看所有规则

查看当前所有防火墙规则及其编号。

### 删除规则（如果需要）

如果配置错误，可以删除规则：

1. 先查看规则编号
2. 删除指定编号的规则

## 配置云服务商安全组

除了服务器内部的防火墙，云服务商还有一层安全组（或防火墙）保护。

### Google Cloud Platform安全组

GCP在创建虚拟机时如果勾选了"允许HTTP流量"和"允许HTTPS流量"，会自动创建防火墙规则。

如果没有勾选，需要手动创建：

1. 进入GCP控制台
2. 找到"VPC网络" → "防火墙规则"
3. 点击"创建防火墙规则"
4. 配置规则：
   - 名称：allow-http-https
   - 目标：网络中的所有实例
   - 源IP范围：0.0.0.0/0（允许所有IP）
   - 协议和端口：tcp:80, tcp:443
5. 点击"创建"

### 阿里云安全组

1. 进入阿里云控制台
2. 找到"云服务器ECS" → "网络与安全" → "安全组"
3. 找到你的实例使用的安全组
4. 点击"配置规则" → "入方向"
5. 确保有以下规则：
   - SSH：端口22，授权对象0.0.0.0/0
   - HTTP：端口80，授权对象0.0.0.0/0
   - HTTPS：端口443，授权对象0.0.0.0/0

如果没有，点击"添加规则"手动添加。

### 腾讯云安全组

腾讯云的配置与阿里云类似：

1. 进入腾讯云控制台
2. 找到"云服务器CVM" → "安全组"
3. 选择实例的安全组
4. 添加入站规则，开放22、80、443端口

## 安全加固（可选但推荐）

为了提高服务器安全性，可以进行以下加固措施。

### 1. 禁用密码登录，只允许密钥登录

#### 编辑SSH配置文件

打开SSH配置文件。

#### 修改以下配置

找到以下行并修改：

- 将 PermitRootLogin yes 改为 PermitRootLogin prohibit-password
  （意思是禁止root使用密码登录，但允许使用密钥登录）

- 将 PasswordAuthentication yes 改为 PasswordAuthentication no
  （意思是完全禁用密码认证）

**注意：** 在修改这些配置之前，确保你已经配置好了SSH密钥登录，否则会把自己锁在外面！

#### 重启SSH服务

修改保存后，重启SSH服务使配置生效。

#### 测试连接

退出当前SSH连接，重新连接测试，应该使用密钥自动登录，不需要输入密码。

### 2. 安装Fail2Ban防止暴力破解

Fail2Ban可以监控登录日志，自动封禁多次登录失败的IP地址。

## 常见问题排查

### 问题1：无法连接SSH

**症状：** 连接超时或拒绝连接

**排查步骤：**
1. 检查服务器是否在运行（云控制台查看）
2. 检查IP地址是否正确
3. 检查云安全组是否开放22端口
4. 检查服务器内UFW是否开放22端口
5. 尝试重启服务器

### 问题2：Nginx启动失败

**症状：** 启动Nginx时报错

**可能原因：**
- 80端口被占用
- 配置文件语法错误

**解决方法：**
- 检查是否有其他程序占用80端口
- 测试Nginx配置文件语法
- 查看详细错误日志

### 问题3：软件安装失败

**症状：** apt install 报错

**解决方法：**
- 先执行 sudo apt update 更新软件包列表
- 检查网络连接是否正常
- 检查磁盘空间是否充足

### 问题4：防火墙配置错误被锁在外面

**症状：** 启用UFW后无法SSH连接

**解决方法：**
- 通过云服务商控制台的Web SSH或VNC连接到服务器
- 禁用UFW或添加SSH规则
- 重启服务器
