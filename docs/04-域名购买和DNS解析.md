## 域名基础知识

### 什么是域名

域名是网站在互联网上的地址，就像房子的门牌号。访客通过域名来访问你的网站，比域名比IP地址更容易记忆。

例如：
- www.google.com
- www.bohenan.com
- github.com

### 域名的结构

一个完整的域名由几部分组成：

以 www.bohenan.com 为例：
- **www**：子域名（Subdomain），可选部分
- **bohenan**：二级域名（Second-Level Domain），这是你注册的主体
- **com**：顶级域名（Top-Level Domain，TLD），域名的类型

### 常见的顶级域名

#### 通用顶级域名

- **.com**：最流行的域名，适合商业网站
- **.net**：原本用于网络服务商，现在通用
- **.org**：原本用于非营利组织，现在通用
- **.info**：信息类网站
- **.io**：科技公司和初创企业流行选择
- **.dev**：开发者和技术网站

#### 国家顶级域名

- **.cn**：中国
- **.us**：美国
- **.uk**：英国
- **.jp**：日本

#### 新兴顶级域名

- **.tech**：技术类
- **.site**：网站
- **.online**：在线服务
- **.app**：应用程序

### 如何选择域名

1. **简短易记**：越短越好，容易拼写和记忆
2. **有意义**：最好与个人品牌或网站内容相关
3. **避免特殊字符**：不要使用连字符、数字（除非必要）
4. **优先.com**：这是最常见的顶级域名，用户习惯
5. **检查可用性**：确保域名没有被注册
6. **检查商标**：确保不侵犯他人商标权

## 百度智能购买域名流程

### 第一步：搜索域名

1. 访问百度智能云
2. 在首页的搜索框中输入你想要的域名（不需要输入.com后缀）
3. 点击"查域名"按钮
4. 系统会显示该域名在各个后缀下的可用性

**结果说明：**
- 绿色勾号：可以注册
- 红色叉号：已被他人注册
- 购物车图标：可以加入清单

### 第二步：选择域名

1. 浏览可用的域名后缀
2. 推荐选择.com后缀（最常用）
3. 点击"加入清单"按钮
4. 可以同时选择多个域名（如.com和.cn）

### 第三步：填写域名信息

1. 点击"结算"按钮
2. 选择购买年限（建议购买1-3年）
3. 填写域名所有者信息：

**个人用户需要填写：**
- 域名持有者类型：个人
- 域名持有者：真实姓名
- 证件类型：身份证
- 证件号码：身份证号码
- 省份/城市：实际居住地
- 详细地址：真实详细地址
- 邮箱：常用邮箱（非常重要，用于接收验证邮件）
- 手机号：常用手机号

**重要提醒：**
- 所有信息必须真实有效
- 邮箱必须能正常接收邮件
- 信息不真实可能导致域名被收回

### 第四步：完成支付

1. 勾选同意服务协议
2. 选择支付方式（支付宝/微信/银行卡）
3. 完成支付

### 第五步：实名认证（必须）

购买域名后必须完成实名认证，否则域名会被锁定无法使用。

1. 支付完成后进入阿里云控制台
2. 找到"域名" → "域名列表"
3. 找到刚购买的域名，状态显示"未实名认证"
4. 点击"实名认证"按钮
5. 上传证件照片：
   - 个人：上传身份证正反面照片
   - 企业：上传营业执照
6. 提交审核

**审核时间：** 通常1-3个工作日内完成审核。

## DNS解析配置

### 什么是DNS解析

DNS（Domain Name System）是域名系统，作用是将域名转换为IP地址。

当用户在浏览器输入域名时：
1. 浏览器向DNS服务器查询域名对应的IP地址
2. DNS服务器返回IP地址
3. 浏览器根据IP地址访问服务器
4. 服务器返回网页内容

### DNS记录类型

#### A记录（最常用）

将域名指向一个IPv4地址。

例如：bohenan.com → 34.80.123.45

#### AAAA记录

将域名指向一个IPv6地址。

#### CNAME记录

将域名指向另一个域名。

例如：www.bohenan.com → bohenan.com

#### TXT记录

文本记录，用于验证域名所有权、配置SPF等。

#### MX记录

邮件服务器记录，用于配置邮箱服务。

---
DNS解析配置

### 第一步：进入域名解析

1. 登录百度云控制台
2. 找到"域名" → "域名列表"
3. 找到你的域名
4. 点击右侧的"解析"按钮

### 第二步：添加A记录（主域名）

这一步将根域名（不带www）指向服务器。

1. 点击"添加记录"按钮
2. 填写记录信息：

**记录配置：**
- 记录类型：选择 A
- 主机记录：输入 @（@表示根域名）
- 解析线路：默认
- 记录值：输入你的服务器公网IP地址
- TTL：10分钟（600秒）

3. 点击"确认"按钮

**效果：** 用户访问 bohenan.com 会指向你的服务器IP。

### 第三步：添加A记录（www子域名）

这一步将www子域名也指向服务器。

1. 再次点击"添加记录"
2. 填写记录信息：

**记录配置：**
- 记录类型：A
- 主机记录：www
- 解析线路：默认
- 记录值：你的服务器公网IP地址
- TTL：10分钟

3. 点击"确认"

**效果：** 用户访问 www.bohenan.com 也会指向你的服务器IP。

### 第四步：验证解析配置

添加完记录后，可以在解析记录列表中看到刚添加的两条记录：

- @ → IP地址
- www → IP地址


## 验证域名解析

配置DNS解析后，需要验证是否生效。

### 方法一：使用ping命令

**Windows系统：**
打开命令提示符，输入：
ping 域名

**Mac/Linux系统：**
打开终端，输入：
ping 域名

如果返回的IP地址是你的服务器IP，说明解析成功。

### 方法二：使用nslookup

**所有系统都可用：**
nslookup 域名

应该显示你的服务器IP地址。

### 方法三：使用dig命令（Linux/Mac）

dig 域名

查看ANSWER SECTION部分，应该显示你的服务器IP。

### 方法四：在线工具

访问以下网站进行检测：

**DNS Checker（推荐）：**
- 网址：dnschecker.org
- 功能：查看全球不同地区的DNS解析结果
- 可以检测解析是否在全球范围内生效

**What's My DNS：**
- 网址：whatsmydns.net
- 功能：类似DNS Checker，界面更简洁

### 方法五：浏览器测试

在浏览器地址栏输入：
http://你的域名

如果能看到Nginx欢迎页面（或者显示连接错误但不是域名无法解析），说明DNS解析已经生效。

## SSL证书申请

### 为什么需要SSL证书

1. **数据加密**：保护用户隐私和数据安全
2. **浏览器信任**：避免浏览器显示"不安全"警告
3. **SEO优势**：Google等搜索引擎优先收录HTTPS网站
4. **必备功能**：某些浏览器API（如地理位置）要求HTTPS

### Let's Encrypt免费SSL证书

Let's Encrypt是一个免费的SSL证书颁发机构，提供：
- 完全免费
- 自动续期
- 受所有主流浏览器信任
- 有效期90天，支持自动续期

### 安装Certbot工具

Certbot是Let's Encrypt官方推荐的证书管理工具。

#### 第一步：安装Certbot

在服务器上安装Certbot和Nginx插件。

#### 第二步：申请SSL证书

使用自动模式申请证书，Certbot会自动配置Nginx。

命令说明：
- --nginx：使用Nginx插件，自动配置
- -d：指定域名，可以添加多个域名

#### 第三步：交互式配置

运行命令后会有几个交互式问题：

**1. 输入邮箱**
输入你的邮箱地址，用于接收证书到期提醒。

**2. 同意服务条款**
输入A同意。

**3. 是否接收EFF的邮件**
输入N不接收（可选）。

**4. 选择HTTP重定向**
- 选项1：不重定向，HTTP和HTTPS都可以访问
- 选项2：自动重定向HTTP到HTTPS（推荐）

输入2，这样所有HTTP请求会自动跳转到HTTPS。

#### 第四步：证书申请成功

如果一切顺利，会看到成功的提示信息，证书已经安装并配置好。

证书文件位置：
- 完整证书链：/etc/letsencrypt/live/域名/fullchain.pem
- 私钥：/etc/letsencrypt/live/域名/privkey.pem

### 设置自动续期

Let's Encrypt证书有效期只有90天，需要定期续期。

#### 测试续期

测试续期命令是否能正常工作，不会真正续期。

如果测试通过，说明自动续期配置成功。

#### 查看自动续期任务

Certbot安装时会自动添加系统定时任务，每天会自动检查证书是否需要续期。

查看定时任务：
systemctl list-timers | grep certbot

#### 手动续期（如果需要）

如果需要立即续期，可以手动执行：
sudo certbot renew

## DNS解析常见问题

### 问题1：域名解析不生效

**症状：** ping域名无法解析或解析到错误的IP

**原因：**
- DNS解析需要时间传播（几分钟到48小时）
- 本地DNS缓存
- 配置错误

**解决方法：**
1. 等待一段时间，通常10-30分钟内生效
2. 清除本地DNS缓存：
   - Windows：ipconfig /flushdns
   - Mac：sudo dscacheutil -flushcache
   - Linux：sudo systemd-resolve --flush-caches
3. 检查DNS配置是否正确
4. 使用在线工具检查全球解析情况

### 问题2：www和不带www解析不一致

**症状：** www.域名能访问，域名不能访问（或反之）

**原因：** 只添加了一条A记录

**解决方法：** 确保同时添加了两条A记录：
- @ → 服务器IP
- www → 服务器IP

### 问题3：SSL证书申请失败

**症状：** Certbot报错无法申请证书

**常见原因：**
- 域名解析未生效
- 80端口未开放或被占用
- 防火墙阻止了验证请求

**解决方法：**
1. 确认域名已正确解析到服务器IP
2. 检查80端口是否开放和可用
3. 检查防火墙和安全组配置
4. 查看详细错误日志

### 问题4：证书过期

**症状：** 浏览器显示证书过期警告

**原因：** 自动续期失败或未配置

**解决方法：**
1. 手动续期证书
2. 检查自动续期任务是否正常
3. 重启Nginx服务
